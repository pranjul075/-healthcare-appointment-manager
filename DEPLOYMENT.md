# Deployment Guide

This app is a plain Node.js server with a SQLite file on disk, so it needs a host that
keeps a persistent filesystem and runs a long-lived Node process. Render's free tier works
well for this. Vercel does not, because it runs your code in short-lived serverless
functions with no persistent disk, so the SQLite file would reset constantly.

## 1. Push the code to GitHub

```
cd healthcare-appointment-manager
git init
git add .
git commit -m "Healthcare appointment manager"
git branch -M main
git remote add origin https://github.com/<your-username>/<your-repo>.git
git push -u origin main
```

Make sure the repository is public, per the submission guidelines. Do not commit `.env`
or the `data/` folder — both are already excluded in `.gitignore`.

## 2. Create a Render account and a new Web Service

1. Go to render.com and sign up (free), then click **New > Web Service**.
2. Connect your GitHub account and select the repository you just pushed.
3. Render will detect the `render.yaml` file in the repo and pre-fill the build command
   (`npm install`), start command (`npm start`), and a 1GB persistent disk mounted at
   `/data`. Accept the suggested settings.
4. If Render does not pick up `render.yaml` automatically, set these manually:
   - Build command: `npm install`
   - Start command: `npm start`
   - Add a Disk: mount path `/data`, size 1GB

## 3. Fill in the environment variables

`render.yaml` already lists every variable the app uses; Render will prompt you to fill in
the ones marked `sync: false` when you create the service. Open the service's
**Environment** tab afterwards if you need to add or change anything.

Required for the app to function at all:

| Variable | Value |
|---|---|
| `DB_PATH` | `/data/app.db` (already set by render.yaml) |
| `JWT_SECRET` | auto-generated by render.yaml, or run `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"` locally and paste the result |
| `ADMIN_EMAIL` | the email you will use to log in as admin |
| `ADMIN_PASSWORD` | a real password, not a placeholder |

Optional, add these to turn on the extra features:

| Variable | Where to get it |
|---|---|
| `ANTHROPIC_API_KEY` | console.anthropic.com > API Keys |
| `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASSWORD`, `MAIL_FROM` | Gmail: use an **App Password** (Google Account > Security > 2-Step Verification > App Passwords), host `smtp.gmail.com`, port `587`. Or use SendGrid / Mailgun's free tier and their SMTP credentials. |
| `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET` | Google Cloud Console, see step 4 below |
| `GOOGLE_REDIRECT_URI` | `https://<your-render-url>.onrender.com/api/calendar/callback` |
| `APP_BASE_URL` | `https://<your-render-url>.onrender.com` |

## 4. Set up Google Calendar OAuth (skip if you don't need calendar sync)

1. Go to console.cloud.google.com, create a project.
2. APIs & Services > Library > enable **Google Calendar API**.
3. APIs & Services > Credentials > Create Credentials > OAuth Client ID > Web application.
4. Under Authorized redirect URIs, add: `https://<your-render-url>.onrender.com/api/calendar/callback`
5. Copy the Client ID and Client Secret into the Render environment variables.

## 5. Deploy

Click **Create Web Service** (or **Manual Deploy > Deploy latest commit** if it already
exists). Watch the logs; you should see:

```
Admin account ready: <your admin email>
Healthcare appointment manager running on port 10000
Background reminder jobs scheduled
```

## 6. Verify it works

1. Visit `https://<your-render-url>.onrender.com/api/health` — should return `{"status":"ok"}`.
2. Visit the root URL, click Sign In, log in with your admin credentials.
3. As admin, add a doctor.
4. Log out, register a patient account, book an appointment, confirm you get a
   confirmation email (if SMTP is configured) and the doctor sees the AI pre-visit summary.

## 7. Submit

Share the GitHub repository link (public, `main` branch) and the live Render URL, per the
assignment's submission guidelines.
